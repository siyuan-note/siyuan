# AI provider logos

The SVG files in this directory are bundled locally so the AI settings UI does not hotlink third-party assets.

The `openai.svg` file is extracted without altering its path geometry from the official [OpenAI Blossom asset](https://images.ctfassets.net/kftzwdyauwt9/3hUGLn3ypllZ0oa01qOYVq/28e8188e6f11b84c3e876569d492734f/Blossom_Light.svg) and normalized to the same 24-by-24 view box as the other provider icons. Use of the logo is subject to the [OpenAI Design Guidelines](https://openai.com/brand/).

The `deepseek.svg`, `minimax.svg`, `aliyun.svg`, `gemini.svg`, `mistral.svg`, `openrouter.svg`, `ollama.svg`, and `lmstudio.svg` files come from [Simple Icons 16.21.0](https://github.com/simple-icons/simple-icons/tree/16.21.0). The `moonshot.svg` file comes from [Simple Icons 16.27.0](https://github.com/simple-icons/simple-icons/blob/16.27.0/icons/moonshotai.svg). Simple Icons is distributed under CC0-1.0, but the depicted logos remain trademarks of their respective owners and should be used according to each brand's guidelines.

The `zhipu.svg` file is extracted from the 24-by-24 `pure-logo.svg` symbol bundled by the official [Zhipu AI Open Platform](https://www.bigmodel.cn/). The `siliconflow.svg` file preserves the path geometry of the single-graphic SVG in the official [SiliconFlow logo package](https://siliconflow.cn/brand) and normalizes its transparent margins to a 24-by-24 view box.

The `volcengine.svg` file comes from [theSVG](https://github.com/glincker/thesvg/blob/aa0605996b4ad4fdda98502f84021b3c3a64847d/public/icons/volcengine/default.svg), distributed under the MIT license. The depicted logo remains a trademark of Volcengine.

Groq uses a generated initial avatar because its [trademark policy](https://groq.com/trademark-policy) limits nominative fair use to its word marks and excludes logos. Custom providers also use generated initial avatars.
